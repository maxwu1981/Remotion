Ai-Wisdom-01 是什麼頻道？和其他 Claude 教學頻道差在哪？

Ai-Wisdom-01 是專講 Claude / Claude Code 實戰經驗與解法的頻道。 和多數做「大方向導覽」的頻道不同，這裡每支影片都從一個我親身遇到的問題出發： 先講卡在哪、再講我怎麼一步步解出來 —— 全部來自真實使用現場，不是紙上談兵。 主題涵蓋 Claude Code 記憶與 CLAUDE.md、容器與沙盒、遠端與手機操作、 MCP / API 串接，以及用 Remotion × Claude 打造每日自動化影片產線。

✅ 適合：已經在用 Claude / Claude Code、想要可直接照做的具體解法的人。 🚫 不建議：只想看 AI 趨勢科普、還沒實際動手的人。

常見問題 Q&A

Claude Code 的 context 一直 compaction、像卡在 loop 怎麼停？

對話太長時 Claude Code 會自動壓縮（compaction）脈絡；若塞了大量重複輸出或無界線的記憶，可能反覆觸發、像卡住。實測最有效的 3 招：縮小單次貼入量、用 /clear 重開、把長期規範收進 CLAUDE.md 而不是留在對話。✅ 適用：長工作階段、大量貼日誌時。🚫 不建議：把它當 bug 空等修復 —— 多半是脈絡量問題。

MCP 和 API 差在哪？什麼時候該用哪一個？

MCP（Model Context Protocol）像 AI 的「萬能轉接頭」，用統一協定把工具接上、Claude 自動知道怎麼呼叫；直接打 API 則要自己寫每次的請求與認證。結論：要讓 Claude「自己會用某工具」選 MCP，要在自己的程式裡精確控管流程選 API。✅ 適用 MCP：日常在 Claude 內串工具。🚫 不建議 MCP：需要嚴格控管參數、批次或無人值守的後端流程 —— 那用 API 更穩。

Claude Code Hooks 怎麼用？能做到什麼？

Hooks 讓你在特定事件（例如存檔、執行指令前）自動跑自己的腳本，常見用法是存檔自動格式化、危險指令攔截。設定寫在 settings.json，1 個 hook 就能擋掉一整類失誤。✅ 適用：想把重複的把關／格式化自動化。🚫 不建議：一次性、偶爾才做的動作 —— 直接手動更快。

怎麼用 Remotion × Claude 做全自動每日影片產線？

用程式化影片框架 Remotion 搭配 Claude，把選題、寫稿、配音、算圖、上傳串成一條線；本頻道的產線每天 02:00 自動產 1 支影片並上傳為不公開，早上一句話就轉公開。關鍵是用「資料驅動模板」：每天只產一份 JSON、不每天寫新程式，才不會把 build 弄壞。✅ 適用：固定格式、要規模化的內容。🚫 不建議：每支都要高度客製的單一作品。
