Claude 本地 vs 雲端完整圖解：容器、沙盒、Cowork、Dispatch 的記憶與檔案怎麼共存？

很多人搞不清楚：在 Claude / Claude Code 裡，東西到底跑在你的電腦、還是雲端？哪些記憶和檔案會共用、哪些永遠碰不到？ 一句話結論：工作「在哪裡執行」決定 3 件事——能碰哪些檔案、用哪一份 CLAUDE.md/記憶、在哪個裝置看得到。 而本地與雲端之間是一道單向牆：本地能上傳、讀取雲端，雲端卻 100% 讀不到你的本機。

本文釐清的實體：本地電腦（你的 Mac）· 沙盒 Sandbox · 容器/臨時容器 Container · Chatbot（Claude.ai）· Cowork · Claude Code（本地 CLI／雲端版）· Dispatch · 橋接層（Google Drive／MCP）· 本機 CLAUDE.md vs 雲端 CLAUDE.md。

Claude 的本地電腦、沙盒、容器、Chatbot、Cowork、Claude Code、Dispatch 各是什麼？

這 7 個名詞其實分成兩個世界。本地（你的 Mac）有 4 個：本地電腦本體、沙盒（本地的權限圍欄）、Claude Code 本地 CLI、Cowork——都跑在你機器上、用本機 CLAUDE.md、只有本地看得到。雲端（伺服器）有 3 個：Chatbot（Claude.ai）、容器／臨時容器、Dispatch——跑在雲端、用雲端 CLAUDE.md、任何裝置都看得到。

👉 一句話：先把 7 個名詞分成「本地 4 個」和「雲端 3 個」，後面全部就通了。

Cowork 可以在手機上執行嗎？

不行。Cowork 是 100% 在你本機（Mac）執行的；手機在雲端側，依單向牆規則連不進你的電腦，所以開不了、也看不到本機正在跑的 Cowork。想在手機上做事，只能改用雲端的 Dispatch 或 Chatbot。

🚫 常見誤解：「手機開 Cowork 會在我電腦跑」——錯，那會破壞單向牆；手機根本碰不到你的 Mac。

Dispatch 和 Claude Code 是什麼關係？

Dispatch 就是「雲端版的 Claude Code」。它在雲端的臨時容器裡跑，讀的是雲端 CLAUDE.md、不是你本機那份。因為它在雲端，你在本地或手機任何裝置都看得到它的進度，但它碰不到你 Mac 上的檔案。

👉 一句話：Claude Code 有兩種——本地 CLI（碰本機）與雲端 Dispatch（碰不到本機、到處可見）。

為什麼雲端讀不到我本機的檔案？（單向牆）

因為雲端沒有讀你電腦的權限——這是一道單向牆：本地可以主動上傳、讀取雲端，雲端不能反向讀本地。所以雲端跑的 Dispatch 與容器，預設 100% 看不到你硬碟上的東西。

🚫 常見誤解：以為「雲端會自動同步我的檔」——不會，要你主動把資料送上去。

記憶（CLAUDE.md）和對話能不能跨本地、雲端共用？

分兩種。記憶 / CLAUDE.md：可以共用，但方式是「本地單向、手動上傳」到雲端；上傳後是獨立副本，兩邊各改會漂移，建議固定以本機那份為唯一真相往上推。對話（context）：不能共用——雲端每次對話都是全新的臨時容器，你只能自己把重點手動搬過去。

👉 一句話：記憶搬得動（單向上傳），對話搬不動（只能手動帶）。

把對話存成檔案，是不是就能上傳共用了？

對，這正是正解。對話的即時 context 本身搬不動；但只要把它固化成檔案（寫進 CLAUDE.md、memory 檔或一份 notes.md），它就變成「檔案／記憶」，立刻適用「可單向上傳共用」那條規則。所以真正的界線不是「對話 vs 記憶」，而是「沒存下來的即時對話（搬不動）」 vs 「已存成檔案的內容（可上傳）」。代價：要手動萃取重點，存的是快照／摘要，不是 100% 還原整段對話。

👉 一句話：存檔這個動作，就是把搬不動的對話變成搬得動的記憶。

本機的東西想讓雲端用，有哪幾種方法？

有 2 種正規路：① push 進 git / repo，雲端版 Claude Code／Dispatch 啟動時會帶你的 repo 快照當起點；② 放進橋接層（Google Drive、MCP 連接器），本機和雲端都可讀。雲端不能直接讀你硬碟，這兩條是把資料送上雲端的唯一通路。

👉 一句話：要嘛 push git、要嘛放 Drive/MCP，沒有第三條直通本機的路。

圖裡的「雲端／伺服器」就是 GitHub 嗎？

不是——這裡有兩朵不同的雲，別混在一起。Anthropic 雲端（圖裡的「伺服器」）是跑 Dispatch／臨時容器／Chatbot 的執行伺服器，你碰不到它內部。GitHub（微軟的雲）是存 code、託管網站（GitHub Pages）的地方，在本文屬於橋接層——你 push 上 GitHub，Anthropic 的雲端容器啟動時再去拉你的 repo。

👉 一句話：GitHub 是資料中轉站（橋接層），不是跑 Claude 的伺服器；兩朵雲、不同公司、不同角色。
